## Module 1 — Disease Detection

**Model:** EfficientNet-V2-S (timm), fine-tuned on PlantVillage
**Checkpoint:** `backend/models/efficientnet_v2s_plantvillage.pth`
**Classes:** 38 (PlantVillage standard — Apple, Tomato, Pepper, etc. + healthy variants)
**Endpoint:** `POST /disease/analyze` (existing, no shape change)
**Expected accuracy:** ~99% top-1 on PlantVillage test set
**Inference:** ~80ms on GPU

**To train:**
```bash
cd .worktrees/backend/backend
python -m ai.disease.train
```
Downloads PlantVillage from HuggingFace, fine-tunes for 20 epochs, saves checkpoint.

**Why V2-S over B0:** 21M params vs 5.3M, ~4× accuracy improvement on this task, still fast.

## Module 2 — Forecasting (PatchTST)

**Model:** PatchTST (IBM Research, ICLR 2023)
**Checkpoints:** `backend/models/patchtst_{metric}.pth` (4 models: temp, humidity, ph, ec)
**Input:** 1,008 readings (7 days × 144 ticks at 10-min resolution)
**Output:** 5 daily forecast values
**Endpoint:** `GET /forecast/{roomId}/{metric}` (existing, no shape change)
**Inference:** <20ms on CPU

**Training phases:**
- Phase 1 (now): synthetic data generated by the simulation engine (~90 virtual days)
- Phase 2 (after 30 days real data): retrain on Firestore readings, auto-runs every 14 days

**To train:**
```bash
python -m ai.forecast.train --phase 1   # synthetic
python -m ai.forecast.train --phase 2   # real Firestore data
```

**Why PatchTST over LSTM:** Transformer patches outperform LSTM/TCN on multivariate time series benchmarks. Small model (~2MB), interpretable attention weights.

## Module 3 — Chatbot (Mistral-7B + RAG)

**LLM:** `mistral:7b-instruct-v0.3-q4_K_M` via Ollama (~4GB VRAM, Q4 quantized)
**RAG:** ChromaDB at `backend/data/chroma/`, embedded with `all-MiniLM-L6-v2`
**Knowledge base:** `backend/ai/chat/knowledge/` — .md files covering pH/EC tables, VPD, diseases, feeding schedules
**Endpoint:** `POST /chat` (existing, no shape change)

**To set up:**
```bash
ollama pull mistral:7b-instruct-v0.3-q4_K_M
python -m ai.chat.rag --build    # builds ChromaDB from knowledge/ files
```

**RAG pipeline per request:**
1. Embed user message → top-3 knowledge chunks (cosine similarity)
2. System prompt = greenhouse persona + live Firestore context + retrieved chunks
3. Last 10 messages appended → Ollama `/api/chat`

**Why Mistral-7B over llama3.2:3b:** Much stronger instruction following and reasoning. Q4 quantization fits in ~4GB VRAM on the RTX 3050.

## Module 4 — Recommendations

**Endpoint:** `POST /recommend/{roomId}` (NEW)
**Frontend:** "Apply AI Recommendations" button in `rooms.$roomId.tsx` Setpoints tab

**Two layers:**

### Rule Engine (always active)
Deterministic metric vs target checks. Fires in <1ms. See `ai/recommend/rules.py`.
Covers: pH up/down dosing, EC flush/dose, temp/humidity HVAC, VPD correction.

### LightGBM Optimizer (activates after 30 days of data)
Features: 7-day rolling stats per metric, stage, day ratio, current setpoints.
Output: delta adjustments to setpoint centers with confidence scores.
Retrains every 14 days via background thread.

**Response shape:**
```json
{
  "immediate": [{"metric": "ph", "action": "Dose 15ml pH Down", "severity": "warning"}],
  "optimizations": [{"setpoint": "temp", "current": [22, 26], "suggested": [23, 27], "confidence": 0.84}],
  "model_active": true,
  "generated_at": "..."
}
```

**To train LightGBM (run after 30 days of data):**
```bash
python -m ai.recommend.train
```
